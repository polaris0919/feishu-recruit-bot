# Boss Guide：Recruit Ops Agent 分阶段使用手册

本文面向招聘流程的最终决策人，说明老板如何在飞书里向 Recruit Ops Agent 下发指令，以及 Agent 通常会如何确认、执行和反馈。

## TL;DR

- **入口**：直接在飞书里发自然语言，不需要记 CLI 命令。
- **真源**：候选人状态以 Recruit Ops 数据库为准，不以聊天记忆为准。
- **权限**：老板可以触发完整招聘流程，包括一面、笔试、二面、Offer、入职、留池、删档和强制修正。
- **确认**：发邮件、建/删日历、删档、强制跳转、Offer / 入职等关键动作，Agent 会先给出拟执行计划，等待确认。
- **安全**：候选人邮件不会自动触发高风险动作。候选人确认时间、要求改期、拒 Offer 或退出流程，都需要老板拍板。
- **最近规则更新**：`reject_delete` 统一走归档删档入口；候选人邮件发送成功后才推进状态；飞书内部通知失败不回滚候选人状态。

---

## 0. 使用模型

老板只需要表达业务目标。Agent 负责：

1. 识别候选人；
2. 查询当前状态；
3. 判断当前阶段允许哪些动作；
4. 给出拟执行动作；
5. 等待确认；
6. 执行并反馈结果。

推荐表达格式：

```text
候选人 + 当前决策 + 必要参数
```

例如：

```text
张三一面通过，发笔试
李四笔试通过，约二面，周五 15:00
王五二面通过，通知 HR 准备 offer
赵六拒 offer，保留人才库
```

不推荐表达：

```text
这个人不要了
都删了
随便安排一下
按你觉得合适的来
```

## 1. 全局确认机制

### 1.1 什么时候会直接执行

只读查询通常会直接执行：

```text
查一下张三
今天有哪些面试？
哪些人在等我拍板？
看一下李四最近的邮件
```

Agent 通常会回复：

```text
[候选人状态]
- 姓名：张三
- 阶段：笔试已审阅
- 当前待办：等待老板决定是否进入二面
- 最近邮件：候选人已提交笔试
```

### 1.2 什么时候必须确认

以下动作通常需要确认：

- 给候选人发邮件；
- 创建或删除飞书日历；
- 拒绝并删除候选人；
- 强制修正候选人阶段；
- Offer / 入职相关关键动作。

Agent 通常会先回复：

```text
[待确认]
候选人：张三（t_xxxxxx）
当前阶段：笔试已审阅
拟执行：发送二面邀请邮件，时间为 5 月 22 日 15:00；邮件成功后推进到二面排期中
风险：邮件会发给候选人

确认请回复：
确认张三笔试通过，约二面，5 月 22 日 15:00
```

你可以确认：

```text
确认
OK
按这个执行
确认张三笔试通过，约二面，5 月 22 日 15:00
```

涉及删除、强制修正时，建议写完整：

```text
确认张三一面不通过，发拒信并删除
确认删除张三 t_xxxxxx
确认强制把张三改到二面排期中，原因是线下已沟通
```

## 2. 模块 A：查询与总览

适用阶段：所有阶段。

### 老板怎么下指令

```text
查一下张三现在到哪一步了
今天有哪些面试？
本周还有谁要面？
哪些人在等我拍板？
列一下还在流程中的候选人
看一下李四最近的邮件
```

### Agent 通常怎么回复

```text
[候选人状态]
- talent_id：t_xxxxxx
- 姓名：张三
- 当前阶段：二面排期中
- 候选人邮箱：zhangsan@example.com
- 下一步：等待候选人确认二面时间；确认后需要老板明确指令创建日历
```

或者：

```text
[今日面试]
1. 张三，一面，15:00，已建日历
2. 李四，二面，17:00，已建日历
```

### 注意事项

- 查询不会改数据，也不会发邮件；
- 如果出现同名候选人，Agent 会要求你指定邮箱或 talent_id；
- 如果你不确定候选人阶段，先查询再决策。

---

## 3. 模块 B：新候选人 / 一面前

典型阶段：新建。

老板通常不负责录入 CV，但可以查询 HR 新录入的候选人，并决定是否安排一面。

### 老板怎么下指令

```text
查一下今天新录入的候选人
看一下张三的简历摘要
给张三安排一面，明天下午三点
```

### Agent 通常怎么回复

首次给一面时间时，Agent 可能先要求确认：

```text
[请确认一面时间]
候选人：张三（t_xxxxxx）
我解析到一面时间：明天 15:00

确认无误请回复：
确认安排张三明天 15:00 一面

在确认前不会发邮件、不会创建日历、不会改阶段。
```

确认后，Agent 通常会回复：

```text
[一面已安排]
- 候选人：张三
- 一面时间：明天 15:00
- 一面邀请邮件：已发送
- 飞书日历：已创建
- 当前阶段：一面已安排
```

### 注意事项

- 时间必须尽量明确；
- 如果 Agent 无法判断面试官，会要求 HR 或老板手动指定；
- 一面安排会涉及候选人邮件和日历，通常需要确认。

---

## 4. 模块 C：一面排期与改期

典型阶段：一面排期中 / 一面已安排。

### 老板怎么下指令

```text
张三一面改到周五上午十点
候选人确认了，就按这个时间建日历
张三说不在国内，先暂缓，等回国再约
张三一面没来，给他一次机会，改到周五上午十点
张三一面 no-show，发 no-show 拒信并删除。确认删除张三 t_xxxxxx
```

### Agent 通常怎么回复

改期前：

```text
[待确认：一面改期]
候选人：张三（t_xxxxxx）
当前一面时间：5 月 20 日 15:00
拟改为：5 月 23 日 10:00
拟执行：
1. 删除旧日历
2. 发送改期邮件
3. 更新候选人状态为一面排期中，等待候选人再次确认

确认请回复：
确认张三一面改到 5 月 23 日 10:00
```

候选人确认时间后：

```text
[待确认：创建日历]
候选人：张三
候选人已确认一面时间：5 月 23 日 10:00
拟执行：创建飞书日历，并将状态更新为一面已安排
```

### 注意事项

- 候选人在邮件里确认时间后，Agent 不会自动建日历；
- 改期可能涉及旧日历删除和新邮件发送；
- 候选人暂缓回国时，Agent 会把候选人挂起，不做拒绝处理；
- 不存在 `ROUND1_NO_SHOW` / `ROUND2_NO_SHOW` 这种候选人阶段。no-show 不是状态，只能选择重排，或发送 no-show 专用拒信后删除归档；
- no-show 不能标记成 `EXAM_REJECT_KEEP`。`EXAM_REJECT_KEEP` 只用于笔试未通过等笔试相关留池，不用于一面/二面缺席；
- 如果要判定 no-show 并删除，请明确写出候选人和确认删除，例如：`张三一面 no-show，发 no-show 拒信并删除。确认删除张三 t_xxxxxx`。

## 5. 模块 D：一面结果

典型阶段：一面已安排。

### 老板怎么下指令

```text
张三一面通过，发笔试
张三一面通过，跳过笔试，直接约二面，下周三下午两点
张三一面不合适，发拒信并删除
```

### Agent 通常怎么回复

一面通过：

```text
[待确认：一面通过]
候选人：张三（t_xxxxxx）
拟执行：发送笔试邀请邮件
执行规则：邮件发送成功后，候选人才会进入“笔试已发送”

确认请回复：
确认张三一面通过，发笔试
```

执行后：

```text
[一面结果已记录]
- 候选人：张三
- 结果：一面通过
- 笔试邀请：已发送
- 当前阶段：笔试已发送
```

一面不通过并删除：

```text
[待确认：拒绝并删除]
候选人：张三（t_xxxxxx）
拟执行：
1. 发送拒信
2. 归档候选人资料和邮件记录
3. 从当前人才库删除

确认请回复：
确认张三一面不通过，发拒信并删除
```

执行后：

```text
[一面结果已记录]
- 候选人：张三
- 结果：未通过，已发拒信并归档删除
```

### 注意事项

- 一面通过时，邮件失败则不会推进到“笔试已发送”；
- 一面不通过通常走拒信 + 归档删除；
- 删除不是普通状态，而是物理归档删除。

---

## 6. 模块 E：笔试发送与评审

典型阶段：笔试已发送 / 笔试已审阅。

### 老板怎么下指令

```text
张三笔试提交了吗？
看一下张三笔试评审
哪些人在笔试已审阅但还没决定？
```

### Agent 通常怎么回复

笔试提交后，Agent 可能推送：

```text
[笔试评审完成]
候选人：张三（t_xxxxxx）
AI 评审：建议通过
理由摘要：
- 代码完整度较高
- 风控思路基本清楚
- 个别边界条件可追问

请老板决定：
1. 笔试通过，约二面
2. 笔试不过，保留人才库
3. 笔试不过，发拒信并删除
```

### 注意事项

- AI 评审只是参考，不会自动决定通过或不通过；
- 长时间未拍板时，Agent 可能通过飞书提醒你；
- 笔试超时未提交的候选人，可能由 cron 自动发拒信并归档删除。

---

## 7. 模块 F：笔试结果

典型阶段：笔试已发送 / 笔试已审阅。

### 老板怎么下指令

```text
张三笔试通过，约二面，周五 15:00
张三笔试不过，保留人才库
张三笔试不过，发拒信并删除
```

### Agent 通常怎么回复

笔试通过：

```text
[待确认：笔试通过]
候选人：张三（t_xxxxxx）
拟执行：发送二面邀请邮件，时间为周五 15:00
执行规则：邮件发送成功后，候选人才会进入“二面排期中”

确认请回复：
确认张三笔试通过，约二面，周五 15:00
```

执行后：

```text
[笔试结果已记录]
- 候选人：张三
- 结果：通过，进入二面阶段
- 二面通知邮件：已发送
- 当前阶段：二面排期中
- 老板飞书日历：暂不创建，待候选人确认后再落盘
```

笔试不过留池：

```text
[笔试结果已记录]
- 候选人：张三
- 结果：笔试未通过，保留人才库
- 当前阶段：笔试未通过（保留）
```

笔试不过删除：

```text
[待确认：拒绝并删除]
候选人：张三
拟执行：发送拒信，并归档删除候选人
```

### 注意事项

- 二面邀请邮件成功后才会推进状态；
- 邮件失败时，候选人仍停留在原阶段；
- 删除路径会先发拒信，再走统一归档删档入口。

## 8. 模块 G：二面排期与确认

典型阶段：二面排期中 / 二面已确认。

### 老板怎么下指令

```text
候选人确认了，就按这个时间建二面日历
张三二面改到下周一上午十点
张三说不在国内，先暂缓，等回国再约
```

### Agent 通常怎么回复

```text
[待确认：创建二面日历]
候选人：张三（t_xxxxxx）
候选人已确认二面时间：5 月 27 日 10:00
拟执行：
1. 创建飞书日历
2. 更新候选人为二面已确认
```

执行后：

```text
[二面已确认]
- 候选人：张三
- 二面时间：5 月 27 日 10:00
- 飞书日历：已创建
- 当前阶段：二面已确认
```

### 注意事项

- 候选人确认时间后，仍需要老板明确同意建日历；
- 二面改期与一面改期类似，可能涉及删旧日历、发改期邮件、重新等待确认。

## 9. 模块 H：二面结果

典型阶段：二面已确认。

### 老板怎么下指令

```text
张三二面通过，通知 HR 准备 offer
张三二面不过，保留人才库
张三二面不过，发拒信并删除
```

### Agent 通常怎么回复

二面通过：

```text
[二面结果已记录]
- 候选人：张三
- 结果：通过
- 当前阶段：Offer 跟进
- HR 通知：已发送
```

如果 HR 飞书通知失败：

```text
[二面结果已记录]
- 候选人：张三
- 结果：通过
- 当前阶段：Offer 跟进

WARN：HR 飞书通知投递失败。候选人状态不回滚，请人工确认 HR 是否收到，或要求 Agent 重发通知。
```

二面不过留池：

```text
[二面结果已记录]
- 候选人：张三
- 结果：二面未通过，保留人才库
```

二面不过删除：

```text
[待确认：拒绝并删除]
候选人：张三
拟执行：发送拒信，并归档删除候选人
```

### 注意事项

- 二面通过后进入 Offer 跟进阶段；
- 飞书通知 HR 是内部提醒，失败不回滚候选人状态；
- 二面不过可以选择留池或删除。

## 10. 模块 I：Offer 跟进

典型阶段：Offer 跟进。

### 老板怎么下指令

```text
给张三发 offer 材料
张三还在考虑，先跟进
用草稿回复张三
这封邮件先不用回
张三拒 offer，保留人才库
张三拒 offer，删除归档
张三已入职，标记完成
```

### Agent 通常怎么回复

候选人来信后：

```text
[候选人来信]
候选人：张三
阶段：Offer 跟进
邮件摘要：候选人询问入职时间和材料
建议草稿：……

你可以回复：
1. 用草稿回复张三
2. 修改草稿后发送
3. 暂不回复
```

拒 Offer 留池：

```text
[待确认：拒 Offer 留池]
候选人：张三
拟执行：将候选人标记为拒 Offer 但保留人才库
```

已入职：

```text
[待确认：标记入职]
候选人：张三
拟执行：标记为已入职
```

### 注意事项

- Offer 阶段 Agent 可以起草回信，但不会自动发送；
- 候选人拒 Offer 后，可以选择留池或删除；
- 标记入职是流程收尾动作，建议确认 HR 已完成入职手续后再执行。

## 11. 模块 J：删除、留池与强制修正

适用阶段：所有阶段，但属于高风险动作。

### 删除

老板指令：

```text
张三一面不合适，发拒信并删除
张三笔试不过，发拒信并删除
张三二面不过，发拒信并删除
张三拒 offer，删除归档
```

Agent 通常会回复：

```text
[待确认：拒绝并删除]
候选人：张三（t_xxxxxx）
拟执行：
1. 发送拒信
2. 归档候选人资料和邮件记录
3. 删除当前人才库记录

确认请回复：
确认张三发拒信并删除
```

### 留人才库

老板指令：

```text
张三笔试不过，保留人才库
张三二面不过，保留人才库
张三拒 offer，保留人才库
```

Agent 通常会回复：

```text
[结果已记录]
- 候选人：张三
- 结果：不继续当前流程，保留人才库
```

### 强制修正

老板指令：

```text
张三线下已经确认一面时间了，强制改到一面已确认，原因是线下已沟通
李四其实已经入职了，标记完成，原因是 HR 已确认入职
```

Agent 通常会回复：

```text
[待确认：强制修正阶段]
候选人：张三
当前阶段：一面排期中
拟修正为：一面已确认
原因：线下已沟通

确认请回复：
确认强制修正张三阶段，原因是线下已沟通
```

### 注意事项

- 删除是物理归档删除，不是普通状态；
- 删除前会先发拒信；
- 留池表示保留候选人，未来可重新激活；
- 强制修正必须说明原因，不要用它绕过正常邮件、日历或删档流程。
- 二面不允许用强制修正直接改到“二面已确认”；必须先发二面邀请，候选人确认后再由老板明确授权建日历。

---

## 12. 模块 K：异常与补充信息

### 找不到候选人

Agent 通常会回复：

```text
未找到候选人。请补充姓名、邮箱或 talent_id。
```

你可以补充：

```text
查一下邮箱是 zhangsan@example.com 的候选人
```

### 同名候选人

Agent 通常会回复：

```text
找到多个候选人：
1. 张三，邮箱 a@example.com，阶段：一面已安排
2. 张三，邮箱 b@example.com，阶段：笔试已发送

请指定要操作哪一个。
```

你可以回复：

```text
是第二个张三，邮箱 b@example.com
```

### 缺少时间

Agent 通常会回复：

```text
需要明确面试时间。请补充日期和时间。
```

你可以回复：

```text
改到 5 月 22 日下午 3 点
```

### 邮件发送失败

Agent 通常会回复：

```text
邮件发送失败，候选人状态未推进。请稍后重试，或确认邮箱、模板和附件是否正确。
```

### 飞书通知失败

Agent 通常会回复：

```text
飞书通知失败，但候选人状态已更新且不回滚。请人工确认 HR / 面试官是否收到，或要求 Agent 重发通知。
```

### 阶段不允许

Agent 通常会回复：

